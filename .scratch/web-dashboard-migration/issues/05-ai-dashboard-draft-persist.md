## Agent Brief

**Category:** enhancement
**Summary:** AI 抽屉把 `dashboard-draft` 暂停事件渲染成图表草稿预览卡，用户确认后以当前 workspace session 直连写入 `dashboard_page`，并恢复 workflow run。

**Current behavior:**
服务端 dashboardAgent 已能产出 `dashboard-draft` 意图（builder-style widget 配置 + explanation），Router workflow 在写操作前 suspend。前端没有渲染、预览或保存路径——草稿到达后用户看不到，"AI 生成图表并持久化"链路断在最后一公里。

**Desired behavior:**
- 抽屉收到携带 dashboard-draft 的暂停事件时渲染草稿卡：标题、explanation 说明文字、真实数据预览——用 D3-02 编译器把 widget 配置编译为参数化 SurrealQL 聚合，以当前会话直连执行（只读），结果喂给 D3-03 的对应 widget 组件渲染。
- 用户确认保存：以当前会话直连写 `dashboard_page`（新建页面或向既有页面追加 widget），随后 resume（write-confirmed），run 完成；保存出的图表与手工 builder 创建的在编辑/删除/刷新行为上完全一致。
- 用户忽略草稿：resume（write-rejected），不产生任何 dashboard 记录。
- 预览查询失败（schema 漂移、聚合不合法）：卡片显示错误并允许忽略，不提供保存入口。
- 普通成员保存受 `dashboard_page` PERMISSIONS 兜底，权限错误显示中文提示，不在前端预判 is_admin。

**Key interfaces:**
- `DashboardDraftIntent`（shared rpc 类型）——草稿卡输入合约；其 widget 配置必须与 D3-02 编译器的输入类型同口径，不允许出现"AI 专用"的第二套 widget 描述。
- `ChatStreamEvent` 暂停事件与 resume 合约——同 AI-006，decision 区分 confirmed / rejected。
- `dashboard_page` 表（widgets 数组）——持久化目标；写入走直连记录创建/更新路径。
- D3-03 widget 组件——预览渲染复用，零 AI 特化分支。

**Acceptance criteria:**
- [ ] 暂停事件携带 dashboard-draft 时，抽屉渲染草稿卡，含 explanation 与真实数据预览。
- [ ] 预览聚合以当前会话直连执行且为只读（无任何写语句）。
- [ ] 确认保存后 `dashboard_page` 出现对应记录（直连 SELECT 断言），dashboard 屏幕可见且可编辑/删除。
- [ ] 忽略草稿后无任何 dashboard 记录产生，run 以 rejected 路径结束。
- [ ] 预览失败时卡片显示错误且无保存入口。
- [ ] 纯逻辑层测试：草稿 → 预览查询参数、草稿 → dashboard_page 写入 payload 的转换（fake 连接断言），不打真 LLM。

**Out of scope:**
- 服务端草稿生成质量与 prompt 调整（dashboardAgent 已落地）。
- 手工 dashboard builder 本体（D3-01~04）。
- raw SQL 草稿通道——V1 仅 builder-style 配置；raw SQL 待与手工 builder 的风险控制一并设计。
- row-patch 提案卡（AI-006）。
