# RR-012 — Web-only 人工检索表面与资源保存路径重设计

## Parent

`.scratch/resource-retrieval/PRD.md`

## Why this exists

资源检索底座已经有一批 pre-pivot issue 标为 done，但其中人工检索表面依赖 Electrobun 独立窗口、可信本地壳、外部 WebView 和主进程 RPC。`web-only-pivot` 接受后，这些执行形态不再成立。

本 issue 用来重新定稿 Web-only 形态下的人工补库闭环，同时保留已验证的领域契约：resource item、research session、ResourceAgent、citation、候选暂停/恢复、embedding profile。

## What needs a decision

1. 已决策：V1 人工检索表面采用 Web app 内 route/panel。
2. 已决策：V1 不在应用内嵌入第三方网页，不引入 iframe / WebView 证据捕获。
3. 已决策：V1 证据采集采用手动粘贴。来源 URL、来源标题、捕获时间和顺序由可信 panel 表单记录。
4. 已决策：资源保存由用户确认后调用后端 SSE 保存动作完成；后端在同一动作中完成向量化和入库，不提供单独 embedding enqueue endpoint。
5. 已决策：入口沿用 `RR-011`，只保留 composer 发送模式中的"搜索资源"，不恢复 AI 抽屉头部的独立 proactive research 图标。

## 2026-05-21 V1 decision

- V1 采用 Web route/panel + 手动粘贴证据。
- 外部网页由用户在普通浏览器环境中自行打开；应用只保存用户粘贴进可信 panel 的证据和来源元数据。
- V1 不嵌入第三方页面，不注入脚本，不读取外部页面 selection，不给第三方页面 SurrealDB token、AI endpoint 或应用内部权限。
- 入口使用 AI composer 的"搜索资源"发送模式；miss / low-confidence 时进入 research route/panel。
- 保存资源必须是用户显式确认动作。确认动作返回 SSE 进度：校验草稿、生成 embedding、写入 resource item / resource embedding、更新 research session、完成。
- 后端写入 SurrealDB 时使用调用者当前 workspace session，不使用 root、service JWT 或员工 session 代写业务资源。
- V1 不做 embedding enqueue / retry / reindex endpoint。向量化或入库失败时，本次保存失败，UI 保留草稿和证据篮，用户可再次点击保存。

## Candidate direction

- Web route/panel 展示 query、resourceType、候选资源、证据篮、资源草稿和完成检索动作。
- 手动粘贴证据是 V1 唯一证据采集路径；selection capture、浏览器扩展、外部页面 iframe 和 WebView 都不在 V1 内。
- ResourceAgent 继续负责检索、候选、citation answer 和草稿生成。
- 用户确认后的保存走后端 SSE 保存动作，因为模型 provider key 和 embedding 生成必须留在后端。
- SSE 保存动作是资源确认专用窄入口，不恢复通用业务 CRUD 代理，不提供单独的 embedding enqueue 入口。

## Agent Brief

Implement the Web-only V1 resource research surface as an application route/panel. Evidence input is manual paste only. On save, call a dedicated user-confirmed SSE action that validates the draft, generates embeddings with server-held provider keys, writes the resource item and embedding record into the current workspace database using the caller's workspace session, updates the research session `createdResourceIds`, and streams progress back to the panel. Do not add a generic CRUD proxy, an embedding enqueue endpoint, a retry endpoint, or a resource-level reindex UI. On failure, return an SSE error event and leave the draft/evidence in the panel so the user can press save again.
