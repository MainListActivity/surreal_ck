# Surreal CK

这是一个以数据库为底层的工作区式表格应用语境：用户在**工作区**中组织**工作簿**，在**工作簿**中维护一个或多个**数据表**，并通过字段、记录、引用、关系与查询来表达业务数据。

## Language

**工作区**:
用户可感知的协作空间，也是工作簿、仪表盘和其他资源的归属边界。
_Avoid_: 租户, Tenant（当表达用户空间时）

**工作簿（Workbook）**:
用户在工作区内创建和持有的一级数据容器，包含一个或多个数据表及其表视图。
_Avoid_: 文档, Doc, Document

**文件夹**:
工作区内用于组织工作簿的层级容器。
_Avoid_: 分类（当表达正式容器时）

**模板**:
用来初始化工作簿结构和样例数据的创建方式，而不是工作区内长期协作的资源。
_Avoid_: 工作簿（当表达已创建资源时）

**数据表**:
工作簿中的一个一等数据单元，底层直接对应一个真实的数据库表。
_Avoid_: Sheet, 智能表, 页签, 子页面

**字段**:
数据表结构中定义的属性，用来约束记录可包含的数据项。
_Avoid_: 列（当表达结构时）

**列**:
字段在某个视图或表格界面中的展示单元，用来承载显示、排序、隐藏和布局。
_Avoid_: 字段（当表达展示时）

**记录**:
数据表中的一条业务数据，是写入、更新、引用和查询的基本对象。
_Avoid_: 行（当表达数据时）

**行**:
记录在某个表格视图中的展示形态，用来承载选择、排序、分组和编辑交互。
_Avoid_: 记录（当表达展示时）

**表视图**:
数据表的展示方式，用来决定同一批记录如何被查看和操作。
_Avoid_: 视图（当表达统计定义时）

**仪表盘视图**:
可持久化、可复用的统计查询定义，用来为仪表盘组件提供标准化结果。
_Avoid_: 视图（当表达数据表展示方式时）

**仪表盘页面**:
工作区级的展示资源，用来编排多个仪表盘视图，可选关联某个工作簿但不从属于它。
_Avoid_: 工作簿页面, 数据表页面

**计算字段**:
值不是手工直接录入，而是由表达式、规则或查询派生出来的字段。
_Avoid_: 公式（当表达正式领域概念时）

**引用**:
字段级关联，表示一条记录通过某个字段指向另一条记录。
_Avoid_: 关系（当表达字段级关联时）

**关系**:
独立建模的业务关联，本身可以拥有属性、方向或生命周期。
_Avoid_: 引用（当表达独立业务关联时）

**查询**:
面向数据表和记录的取数与计算动作，是用户操作数据的核心方式。
_Avoid_: 过滤（当表达更完整的取数动作时）

## Relationships

- 一个 **工作区** 包含多个 **工作簿**
- 一个 **工作区** 也承载文件夹、仪表盘页面和仪表盘视图等资源
- 一个 **文件夹** 当前只用于组织 **工作簿**
- 一个 **模板** 可以用来创建一个新的 **工作簿**
- 一个 **工作簿（Workbook）** 包含一个或多个 **数据表**
- 一个 **数据表** 直接对应一个底层数据库表
- 一个 **数据表** 在领域上必须隶属于某个 **工作簿**
- 一个 **数据表** 定义多个 **字段**
- 一个 **数据表** 包含多条 **记录**
- 一个 **列** 总是对应一个 **字段**，但只存在于具体展示上下文中
- 一个 **行** 总是对应一条 **记录**，但只存在于具体展示上下文中
- 一个 **数据表** 可以有多种 **表视图**
- 一个 **仪表盘页面** 可以编排多个 **仪表盘视图**
- 一个 **仪表盘视图** 可以被一个或多个仪表盘页面复用
- 一个 **计算字段** 是 **字段** 的一种
- 一个 **引用** 通过 **字段** 把一条 **记录** 指向另一条 **记录**
- 一个 **关系** 连接两条或多条 **记录**，并且可以承载自身属性
- **筛选**、**排序**、**分组** 和 **聚合** 都是 **查询** 的具体形式
- 一个 **仪表盘视图** 通过 **查询** 产生可展示的统计结果

## Flagged ambiguities

- “文档”曾被用来指代 **工作簿（Workbook）**；已定稿：领域规范词使用 **工作簿**，“文档”只视为历史遗留 UI 别名。
- “工作区” 不只是权限容器；已定稿：它既是资源归属边界，也是用户可感知的协作空间。
- “文件夹” 当前只用于组织 **工作簿**；已定稿：不预设它组织其他资源。
- “模板” 当前不是工作区内一等资源；已定稿：它只是创建 **工作簿** 的方式。
- “仪表盘页面” 不从属于 **工作簿**；已定稿：它是工作区级资源，只可选关联工作簿。
- “Sheet” 不应被理解为纯 UI 页签；已定稿：规范词为 **数据表**，它在本产品里直接对应一个真实数据库表。
- 用户不直接管理裸数据库表；已定稿：**数据表** 必须通过 **工作簿** 进入和管理。
- “智能表” 不进入领域规范词；已定稿：它只可作为产品文案别名，不能替代 **数据表**。
- “字段” 与 “列” 不可混用；已定稿：**字段**是结构概念，**列**是展示概念。
- “记录” 与 “行” 不可混用；已定稿：**记录**是数据概念，**行**是展示概念。
- “视图” 不能单独使用；已定稿：数据表展示方式叫 **表视图**，统计定义叫 **仪表盘视图**。
- “计算字段” 是正式概念；已定稿：它是 **字段** 的一种，上位于具体的计算表达方式。
- “公式” 已被彻底禁用；已定稿：统一使用 **计算字段**，不再用“公式”指代任何正式领域概念。
- “引用” 与 “关系” 不可混用；已定稿：**引用**是字段级关联，**关系**是独立建模的业务关联。
- **关系** 的页面交互形态尚未定稿；当前只确定其领域语义，不假定已有稳定 UI 建模方式。
- 当前正式交互只暴露 **引用**；已定稿：**关系** 先保留为领域概念和底层能力，待图关系 UI 成熟后再升级为显式用户能力。
- “查询” 是一级领域概念；已定稿：**筛选**、**排序**、**分组**、**聚合** 都属于 **查询**。
- 当前不把权限机制展开成领域术语；已定稿：只保留 **工作区** 的归属与访问边界，不在本文件展开角色与权限模型。

## Example dialogue

> **开发者**: “我想在这个工作簿里新增一个表，再加几列字段。”
>
> **领域专家**: “这里要说新增一个**数据表**，并为它定义几个**字段**。如果你只是调整界面展示，那改的是**列**，不是字段。”
>
> **开发者**: “那我在仪表盘里做一个按状态分组的统计视图，是不是也算视图？”
>
> **领域专家**: “那是**仪表盘视图**，不是**表视图**。前者是可复用的统计查询定义，后者是数据表的展示方式。”
